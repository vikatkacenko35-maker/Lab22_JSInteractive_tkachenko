# Калькулятор на JavaScript

## 1. Описание проекта

Проект представляет собой интерактивное веб-приложение - калькулятор с графическим интерфейсом, реализованный на чистом JavaScript с использованием HTML и CSS. Калькулятор выполняет базовые арифметические операции и имеет защиту от некорректного ввода.

## 2. Функциональность

### Поддерживаемые операции
- Сложение (`+`)
- Вычитание (`-`)
- Умножение (`*`)
- Деление (`/`)
- Десятичные дроби (через точку `.`)
- Очистка всего выражения (кнопка `C`)

### Ограничения
- Невозможно ввести два оператора подряд (например, `5++3` или `5*/3`)
- Выражение не может заканчиваться оператором
- Точка может присутствовать только один раз в текущем числе (нельзя ввести `5.2.3`)
- Если выражение пустое, нельзя ввести оператор (кроме унарного минуса для отрицательных чисел)
- Запрещено деление на ноль (обрабатывается как ошибка)

### Обработка ошибок
- При некорректном выражении (например, `10/0`) на дисплее отображается сообщение **"Ошибка"**
- Состояние калькулятора сбрасывается после ошибки
- При возникновении ошибки все последующие нажатия (кроме `C`) автоматически очищают состояние
- Встроенная валидация предотвращает ввод заведомо некорректных выражений

## 4. Логика работы

### Последовательность действий при нажатии кнопок

1. **Пользователь нажимает на кнопку калькулятора**
2. Срабатывает обработчик события `click`, который вызывает `handleButtonClick(value)`
3. В зависимости от типа кнопки:
   - **`C`** → очистка выражения и дисплея
   - **`=`** → вычисление результата
   - **Цифра, оператор или точка** → добавление в выражение (с валидацией)
4. После каждого действия вызывается `updateDisplay()` для обновления экрана

### Как работает валидация ввода?

| Ситуация | Действие |
|----------|----------|
| Ввод оператора и последний символ - тоже оператор | Замена последнего оператора на новый (например, `5+*` → `5*`) |
| Ввод точки, но в текущем числе уже есть точка | Игнорирование нажатия |
| Ввод оператора при пустом выражении и оператор не минус | Игнорирование нажатия |
| Ввод любой корректный символ | Добавление в выражение |

### Как происходит вычисление?

1. **Проверка валидности** - `isValidExpression()` удаляет пробелы и проверяет:
   - Отсутствие двух операторов подряд
   - Отсутствие оператора в конце выражения
2. **Безопасное вычисление** - `safeEvaluate()`:
   - Вызывает `isValidExpression()`
   - Если выражение корректно, выполняет `eval(expr)`
   - При ошибке выбрасывает исключение
3. **Обработка результата** - `calculate()`:
   - Проверяет, что результат - конечное число (`isFinite()`)
   - Сохраняет результат в `expression` для продолжения вычислений
   - При ошибке вызывает `showError("Ошибка")`

# Крестики-нолики (Tic-Tac-Toe)

## 1. Описание игры

### Правила игры
- Игровое поле представляет собой сетку 3×3 (9 клеток)
- Первый игрок играет крестиками (`X`), второй — ноликами (`O`)
- Игроки по очереди ставят свой символ в пустую клетку
- Цель игры — первым выстроить три своих символа в ряд: по горизонтали, вертикали или диагонали
- Если все 9 клеток заполнены, а победитель не определён, объявляется ничья

### Логика победы/поражения
- Победа наступает, когда один из игроков первым собирает три своих символа в выигрышную комбинацию
- После победы игра останавливается, дальнейшие ходы невозможны
- Поражение — это ситуация, когда противник собрал выигрышную комбинацию

### Особенности реализации
- Игра полностью реализована на чистом JavaScript без использования сторонних библиотек
- Интерфейс построен с использованием CSS Grid для игрового поля
- После завершения игры (победа или ничья) клики по клеткам игнорируются
- Кнопка «Начать заново» позволяет начать новую партию без перезагрузки страницы

## 2. Игровая механика

### Как определяется победитель?
- В коде определён массив `winConditions`, содержащий все 8 возможных выигрышных комбинаций (3 горизонтали, 3 вертикали, 2 диагонали)
- После каждого хода функция `checkResult()` проверяет, совпадает ли содержимое клеток хотя бы в одной из комбинаций
- Если победная комбинация найдена, игра останавливается и выводится сообщение о победителе

### Как обрабатываются клики?
- На каждую клетку игрового поля через `addEventListener` навешивается обработчик `handleCellClick`
- При клике определяется индекс клетки через атрибут `data-index`
- Перед заполнением клетки проверяется: пуста ли клетка и активна ли игра
- Если условия выполнены, клетка заполняется символом текущего игрока, после чего вызывается `checkResult()`

### Как работает перезапуск?
- Функция `restartGame()` сбрасывает состояние игры:
  - Текущий игрок становится `X`
  - Массив `board` очищается (все элементы становятся пустыми строками)
  - Флаг `gameActive` устанавливается в `true`
  - Текст статуса обновляется на «Ход игрока: X»
  - Все клетки очищаются визуально (удаляется текст `X` и `O`)

## 3. Структура данных

### Массив board
```js
let board = ["", "", "", "", "", "", "", "", ""];
```
Массив из 9 элементов, каждый элемент соответствует одной клетке игрового поля

Индексы от 0 до 8 соответствуют клеткам:

0 1 2 — первая строка

3 4 5 — вторая строка

6 7 8 — третья строка

Если клетка пуста — в массиве хранится пустая строка ""

Если клетка занята — хранится символ "X" или "O"
const winConditions = [
    [0, 1, 2], // верхняя горизонталь
    [3, 4, 5], // средняя горизонталь
    [6, 7, 8], // нижняя горизонталь
    [0, 3, 6], // левая вертикаль
    [1, 4, 7], // центральная вертикаль
    [2, 5, 8], // правая вертикаль
    [0, 4, 8], // главная диагональ
    [2, 4, 6]  // побочная диагональ
];
Каждый вложенный массив содержит индексы трёх клеток, образующих выигрышную линию

Всего 8 выигрышных комбинаций
## Алгоритмы
1. Установить флаг roundWon = false
2. Для каждой комбинации из winConditions:
   - Получить индексы клеток [a, b, c]
   - Если board[a] не пуст И board[a] равен board[b] И board[a] равен board[c]:
     - Установить roundWon = true
     - Прервать цикл
3. Если roundWon == true:
   - Вывести сообщение о победе
   - Установить gameActive = false
   - Выйти из функции

## Алгоритм определения ничьи
1. Проверить, есть ли в массиве board пустая строка ("")
2. Если пустых строк нет (все клетки заполнены):
   - Вывести сообщение "Ничья!"
   - Установить gameActive = false
   - Выйти из функции
3. Если игра не окончена (нет победы и нет ничьи):
   - Переключить игрока
   - Обновить статус
## Алгоритм смены игрока
1. Проверить значение currentPlayer
2. Если currentPlayer == "X":
   - Установить currentPlayer = "O"
3. Если currentPlayer == "O":
   - Установить currentPlayer = "X"
4. Обновить текст статуса: `Ход игрока: ${currentPlayer}`
